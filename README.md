# Проект: пользовательская "Корзина" для `unlink()` в Linux

Проект реализован на языке **C** и состоит из двух частей:

1. `libtrashhook.so` — динамическая библиотека-перехватчик системных вызовов удаления файлов.
2. `trash-util` — консольная утилита для управления корзиной (просмотр, восстановление, очистка, статистика).

Основная идея: вместо необратимого удаления файл переносится в корзину пользователя:

- каталог файлов: `~/.local/share/Trash/files`
- каталог метаданных: `~/.local/share/Trash/info`

Формат метаданных совместим по структуре с подходом FreeDesktop Trash Info (`.trashinfo`), плюс добавлены поля прав/владельца/времени для более точного восстановления.

---

## 1. Состав проекта

- `trash_hook.c` — перехват `unlink()`, `unlinkat()` и `remove()`.
- `trash_util.c` — CLI-утилита `trash-util`.
- `trash_common.c`, `trash_common.h` — общие функции (пути, кодирование, метаданные, расчет размера).
- `Makefile` — сборка.

## 4. Как работает перехват

Библиотека подгружается через `LD_PRELOAD`, подменяя стандартные функции удаления:

- `unlink(path)`
- `unlinkat(dirfd, path, flags)` (без `AT_REMOVEDIR`)
- `remove(path)` (для файлов; директории удаляются стандартно через `rmdir`)

Логика:

1. Определяется абсолютный путь удаляемого файла.
2. Если файл в домашнем каталоге пользователя, выполняется "мягкое удаление":
   - попытка `rename()` в `~/.local/share/Trash/files/<id>`
   - при `EXDEV` (другая ФС): копирование + реальное удаление оригинала
3. Создается метафайл `~/.local/share/Trash/info/<id>.trashinfo`:
   - исходный путь
   - дата удаления
   - mode/uid/gid
   - atime/mtime

Если файл **не в `$HOME`**, используется обычное удаление (без корзины), чтобы не ломать системные сценарии.

## 5. Быстрый запуск

### 5.1 Разовый запуск команды через перехватчик

```bash
LD_PRELOAD="$PWD/libtrashhook.so" rm some_file.txt
```

### 5.2 Запуск shell-сессии с перехватом

```bash
export LD_PRELOAD="$PWD/libtrashhook.so"
bash
```

Все процессы внутри этой сессии будут использовать перехватчик удаления.

### 5.3 Отключение перехвата временно

Есть флаг bypass:

```bash
TRASH_HOOK_DISABLE=1 rm file.txt
```

Или:

```bash
TRASH_HOOK_DISABLE=1 your_program
```

## 6. Руководство по `trash-util`

### 6.1 Просмотр содержимого корзины

```bash
./trash-util list
```

Вывод:

- `ID`
- `Дата удаления`
- `Исходный путь`

`ID` используется для восстановления.

### 6.2 Восстановление файла

```bash
./trash-util restore <ID>
```

Поведение:

- пытается вернуть файл в исходный путь;
- если родительских директорий нет — создаются;
- если по исходному пути уже есть файл — создается новое имя с суффиксом (`.1`, `.2`, ...);
- после восстановления применяются права, владелец и временные метки из метаданных;
- `.trashinfo` удаляется.

Пример:

```bash
./trash-util restore 1713191111_123456789_1234567890
```

### 6.3 Полная очистка корзины

```bash
./trash-util purge
```

Удаляет все из:

- `~/.local/share/Trash/files`
- `~/.local/share/Trash/info`

и пересоздает служебные каталоги.

### 6.4 Статистика корзины

```bash
./trash-util info
```

Показывает:

- количество файлов
- суммарный размер в байтах

## 7. Полный пример использования

```bash
# 1) Сборка
make

# 2) Создаем тестовый файл
echo "test data" > /tmp/demo-trash-file.txt

# 3) Удаляем с перехватчиком
LD_PRELOAD="$PWD/libtrashhook.so" rm /tmp/demo-trash-file.txt

# ВАЖНО: /tmp не относится к $HOME, поэтому файл удалится обычным способом.
# Для демонстрации корзины используйте путь внутри домашнего каталога:
echo "test data" > "$HOME/demo-trash-file.txt"
LD_PRELOAD="$PWD/libtrashhook.so" rm "$HOME/demo-trash-file.txt"

# 4) Смотрим корзину
./trash-util list

# 5) Восстанавливаем
./trash-util restore <ID_из_list>

# 6) Проверяем статистику
./trash-util info
```

## 8. Тестирование (рекомендуемый чек-лист)

1. **Базовое удаление:**
   - создать файл в `$HOME`
   - удалить через `LD_PRELOAD`
   - убедиться, что файл появился в `Trash/files`, а мета — в `Trash/info`

2. **Восстановление:**
   - `trash-util restore <ID>`
   - проверить содержимое файла, права, время модификации

3. **Конфликт при восстановлении:**
   - до восстановления создать файл в исходном пути
   - убедиться, что восстановление происходит в альтернативное имя (`.1`, `.2`)

4. **Очистка:**
   - `trash-util purge`
   - убедиться, что корзина пуста

5. **Статистика:**
   - проверить, что `info` показывает корректное число файлов и размер

## 9. Ограничения текущей версии

- Перехватываются удаления файлов, но не рекурсивная логика для директорий (`rm -r` работает по обычным правилам на уровне вызовов директорий).
- Не реализована отдельная база индексов: `ID` формируется из времени + хэша пути.
- Для упрощения безопасности в этой версии в корзину переносятся только файлы из `$HOME`.
- Утилита не спрашивает подтверждение при `purge` (можно добавить при необходимости).
