# Гайд: как протестировать программу через меню

Этот гайд предназначен для демонстрации работы проекта **без ручного ввода длинных команд `trash-util list/restore/...`**.  
Вы будете работать в основном через интерактивное меню `./trash-util`.

---

## 1. Подготовка проекта

Откройте терминал и выполните:

```bash
cd ~/tar_working_dir/kursach
make clean
make
```

Появятся файлы:

- `libtrashhook.so` — библиотека перехвата
- `trash-util` — утилита с меню

---

## 2. Подготовка тестовых данных

Создайте тестовую папку и очистите старые файлы:

```bash
mkdir -p "$HOME/test_area"
rm -f "$HOME/test_area/file_a.txt" "$HOME/test_area/file_b.txt" "$HOME/test_area/file_b.txt.1"
```

Создайте два тестовых файла:

```bash
echo "alpha" > "$HOME/test_area/file_a.txt"
echo "beta"  > "$HOME/test_area/file_b.txt"
ls -la "$HOME/test_area"
```

Проверьте, что видно `file_a.txt` и `file_b.txt`.

---

## 3. Очистка корзины перед демонстрацией

Чтобы демонстрация была "с нуля":

```bash
./trash-util purge
```

---

## 4. Мягкое удаление файлов (через LD_PRELOAD)

Удалите файлы командой `rm`, но с подключенной библиотекой:

```bash
LD_PRELOAD="$PWD/libtrashhook.so" rm "$HOME/test_area/file_a.txt"
LD_PRELOAD="$PWD/libtrashhook.so" rm "$HOME/test_area/file_b.txt"
```

Важно:

- используйте именно `"$PWD/libtrashhook.so"`;
- не используйте `"$HOME/libtrashhook.so"` (там обычно нет библиотеки).

---

## 5. Запуск меню

Откройте меню:

```bash
./trash-util menu
```

Вы увидите пункты:

1) Показать содержимое корзины  
2) Восстановить файл по ID  
3) Показать статистику  
4) Очистить корзину  
0) Выход

---

## 6. Сценарий теста через меню (рекомендуемый)

### Шаг 6.1 — Проверка содержимого корзины

В меню нажмите `1`.

Ожидаемо:

- в списке 2 элемента;
- у каждого есть `ID`, дата удаления и исходный путь.

### Шаг 6.2 — Проверка статистики

В меню нажмите `3`.

Ожидаемо:

- `Файлов в корзине: 2`
- размер больше 0 байт.

### Шаг 6.3 — Восстановление первого файла

1. Снова нажмите `1` и скопируйте `ID` первого файла.
2. Нажмите `2`.
3. Вставьте `ID` и нажмите Enter.

Ожидаемо:

- сообщение `Восстановлено: ...`.

### Шаг 6.4 — Восстановление второго файла

1. Нажмите `1`.
2. Скопируйте оставшийся `ID`.
3. Нажмите `2`.
4. Вставьте `ID`.

Ожидаемо:

- второй файл тоже восстановится.

### Шаг 6.5 — Проверка, что корзина пуста

Нажмите `1` и затем `3`.

Ожидаемо:

- `Корзина пуста.`
- `Файлов в корзине: 0`

### Шаг 6.6 — Выход

Нажмите `0`.

## 7. Дополнительная проверка конфликта имен (через меню)

Этот шаг показывает, что программа корректно обрабатывает конфликт при восстановлении.

1. Снова создайте `file_b.txt`, удалите его через `LD_PRELOAD`.
2. Откройте меню `./trash-util menu`.
3. Нажмите `1`, скопируйте `ID`.
4. В другом терминале заранее создайте файл по исходному пути:

```bash
echo "conflict-placeholder" > "$HOME/test_area/file_b.txt"
```

5. Вернитесь в меню, нажмите `2`, введите `ID`.

Ожидаемо:

- программа не перезапишет существующий файл;
- выведет сообщение вида `Конфликт: восстановлено как ...file_b.txt.1`.

Проверка:

```bash
ls -la "$HOME/test_area"
```

Должны быть:

- `file_b.txt` (ваш существующий)
- `file_b.txt.1` (восстановленный из корзины)

2. "Создаю тестовые файлы."

4. "Через меню показываю список удаленных файлов и статистику."
5. "Через меню восстанавливаю файлы по ID."
6. "Показываю, что корзина очищается и поддерживается обработка конфликтов."

## 9. Частые ошибки

- Ввели `./trash-util purge~` вместо `./trash-util purge`.
- Указали неправильный путь для `LD_PRELOAD`:
  - правильно: `LD_PRELOAD="$PWD/libtrashhook.so"`.
- Вставка в терминал содержит мусорный префикс вида `^[[200~`.

Если появилась странная команда, нажмите `Ctrl+C` и введите ее вручную.
